AI Automation for End-to-End Test Generation

We want AI to help generate end-to-end tests for Deltek's TIP product, starting with the Nonconformance submodule.

Slide 1

Problem

Writing tests for TIP is still a manual process, and it takes a lot of product knowledge to do it well.

  • Each test has to match the UI, the code, and the internal test format.
  • That makes test creation slow and hard to scale.
  • Deltek proposed we start with a submodule of the TIP codebase (the Nonconformance module) and then generalize it to other modules.

In plain English

We want AI to draft the tests, so that engineers don't have to come up with the tests from scratch.

Slide 2

Solution

We are implementing an agentic workflow that grounds test generation in frontend, backend, database, object-store, framework, and system-test evidence.

  • An ingestion agent reads source materials and creates structured context files for each subsystem.
  • An orchestrator agent uses those context files with a memory layer to plan comprehensive test coverage.
  • Generation agents produce end-to-end tests by referencing the right context files and the relevant code paths.
  • Happy-path descriptions and existing system-test examples are used as guardrails for quality and format.

Slide 3

Biggest Hurdle

The biggest challenge is moving from a hardcoded workflow to a real multi-agent backend that works well in practice.

  • We need multiple agents that are well coordinated, not just one model doing everything.
  • We also want the system to generate stronger tests, not only simple happy paths.
  • That means producing edge cases and checks that actually challenge the product.
